Инструкция по использованию Merger 111

-Поместить текстовые (*.txt) файлы,которые необходимо отсортировать и объеденить в дирикторию */<users>/<username>/<file.txt> 

(Пример C:\Users\home\myFile.txt)

-В командной строке перейти в дирикторию с расположением исполняемого файла 
(Пример D:\IntellijIDEA\Merger)

-Исходя из необходимого метода сортировки требуется составить запрос для запуска программы:

	Если текстовый файл состоит из натуральных чисел,то в качестве аргумента передать -i;

	Если текстовый файл состоит из строчных символов, то в качестве аргумента передать -s;

	Если необходимо отсортировать входные данные "от большего к меньшему",то в качестве аргумента 	передать "-d";

	Если необходимо отсортировать входные данные "от меньшего к большему",то в качестве аргумента 	передать "-a";
 
	Если параметр направления сортировки не указан,то по умолчанию будет передан параметр "-a";

Шаблон запроса :

java -jar <ProgramName>.jar <param1> <param2> <outputFileName.txt> <inputFileName.txt> <inputFileName2.txt>

Где,
	param1 - это параметр направления сортировки -d/-a;
	param2 - это параметр отвечающий за тип данных требующих сортирвки -i/-s;
	("-i" - тип данных int; "-s" - тип данных String)
	outputFileName.txt - имя файла в который нужно будет поместить объединённые данные;
	inputFileName.txt - имя файла требующего сортировки и обьединения;

-Имя выходного файла идёт перед именами файлов требующих слияния.

-Для сортировки более чем 2 файлов, в запросе нужно указать имя файлов через пробел. 

-Выходной файл необходимо разместить в той же дириктории,что и файлы требующие сортировки. В случае отсутствия выходного файла он будет создан автоматически по завершению программы.

Пример готового запроса:

java -jar Merger-1.0-SNAPSHOT.jar -d -i out.txt file1.txt file2.txt file3.txt
